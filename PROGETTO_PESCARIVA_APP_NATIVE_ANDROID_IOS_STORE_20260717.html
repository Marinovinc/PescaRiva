€ Strategia di lancio: vendita diretta, senza store

La chiave: PescaRiva è già una PWA, e la PWA si installa dal browser (Aggiungi a schermata Home) sia su iOS che Android → icona propria, schermo intero, offline. Nessuno store, nessuna commissione.

Incasso diretto — due strade

A · Merchant of Record SCELTO — Lemon Squeezy

Vende chiavi di licenza, gestisce IVA e fatture per te, ~5%, con License API validabile dal client. Non serve un nostro server per i codici a pagamento.

B · Stripe / PayPal diretto (non scelto)

Commissione più bassa (~3% + €0,25), ma gestisci tu l'IVA (OSS UE) e le fatture. Massimo margine, più adempimenti.

🔭 Dopo (opzionale): quando vuoi diffusione di massa, puoi anche pubblicare la versione Capacitor sugli store (§§4-9) mantenendo la vendita diretta. Le due cose convivono: chi compra diretto usa il codice, chi arriva dallo store usa l'IAP. Limiti onesti PWA iOS: nessun listing store (scoperta via sito/FB), alcune API web limitate — adeguato per un pubblico di nicchia.

🔒 App autonoma — mappe e batimetria sul telefono, server solo per l'accesso

Requisito: l'app funziona senza un server di supporto. Le mappe e la batimetria arrivano dai portali pubblici e sono elaborate sul telefono; il nostro backend serve solo a validare il codice di accesso. Se il server è irraggiungibile, l'app continua a funzionare (entro il periodo di grazia e con le zone già visitate in cache).

🔑 Accesso con codice (offline-tolerante)

1
Installa dagli store
2
Inserisci il codice una volta sola
3
Il server firma un token legato al device, con scadenza
4
Funziona offline token verificato in locale · ricontrollo periodico con periodo di grazia

Token firmato dal server e verificato in locale dall'app (chiave pubblica): niente rete per l'uso quotidiano, impossibile falsificarlo. Legato a un UUID casuale del device — nessuna password, nessun dato personale.

📲 Funzionamento autonomo sul dispositivo

L'app non scarica pacchi dal nostro server. Quando l'utente cambia mappa o zona di pesca, l'app scarica da sola e in silenzio ciò che serve direttamente dai portali pubblici e calcola la batimetria SDB sul telefono.

🌐 Download silente dai portali 🛰️ SDB calcolata sul telefono 💾 Cache automatica per l'offline

Ciò che l'utente visita resta in cache: quella zona torna disponibile offline senza operazioni manuali. Nessuna pre-elaborazione su PC, nessun pacco dal nostro server.

Il nostro server NON serve né mappe né batimetria: l'app li prende dai portali pubblici e calcola tutto sul dispositivo. Il backend valida solo il codice d'accesso; l'app si distribuisce dagli store.
🧰 Stack consigliato (serverless, piano gratuito iniziale): una piccola License function su Cloudflare Workers (o Supabase / Firebase) + un DB codici (KV/SQL). Nessuno storage di mappe da gestire — i dati arrivano dai portali. Costi minimi, niente server da mantenere. Si trasmettono solo un codice e un UUID casuale.

5 Requisiti dei due store

🤖 Google Play

  • Account Play Console (25 USD una tantum)
  • Application ID univoco (es. it.pescariva.app)
  • Chiave di firma (Play App Signing)
  • Target API level aggiornato
  • Data safety: nessun dato raccolto
  • Content rating (PEGI 3)
  • Icona 512, feature graphic 1024×500, screenshot
  • Privacy policy URL
  • Upload AAB firmato (test → produzione)

App Store

  • Apple Developer Program (99 USD/anno)
  • Bundle ID (es. it.pescariva.app) + firma via Xcode
  • Stringa uso posizione (NSLocationWhenInUse…)
  • Privacy labels: "Data Not Collected"
  • Icona 1024, screenshot nelle taglie richieste
  • Attenzione regola 4.2 (no "solo sito")
  • Invio review + gestione rilievi
  • Serve un Mac (o cloud-Mac) per compilare

6 Privacy — un punto di forza

PescaRiva è privacy-friendly, e questo semplifica entrambi gli store:

🚫 Nessun server proprio 🚫 Nessun dato personale raccolto 📱 Catture/località solo sul dispositivo 📍 GPS non lascia il telefono

Si contattano servizi cartografici di terze parti (Esri, OSM, EMODnet, Copernicus/Sentinel, Overpass) solo per scaricare mappe/dati. Serve pubblicare una privacy policy (una pagina sul sito attuale) e linkarla nei due store.

✅ Bozza PRONTA: privacy.html (pubblicabile a /PescaRiva/privacy.html). Da completare solo titolare, indirizzo ed email di contatto; versione inglese completa con la localizzazione.
